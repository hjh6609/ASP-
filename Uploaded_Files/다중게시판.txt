다중게시판이라고 하면 말그대로 게시판 소스를 가진 페이지는 하나이지만 

여러개의 게시판을 생성할수 있는것을 말합니다.


예를들어 제로 보드 같은 경우 다중게시판에 속합니다.

사용자가 게시판을 생성하고 싶은 만큼 생성할수 있습니다.

더 쉽게 말하자면

커뮤니티 사이트 같은곳의 게시판은 모두 다중게시판입니다.

예전에는 게시판을 만들때 하나의 DB테이블을 가지고 하나의 게시판만을 생성해서 사용했지만

지금은 커뮤니티 사이트처럼 게시판을 만들고 싶으면 자동적으로 게시판을 추가 할때

DB테이블이 생성되어 게시판이 하나더 생기는 것입니다.

(게시판 소스페이지는 하나!! 데이타베이스에 테이블만 추가)

즉, 다중게시판이라고 하면 사용자가 게시판을 생성하고자 한다면

DB(데이타베이스)에 자동으로 게시판의 내용을 가진 테이블이 생성되고

또한 게시판까지 추가 됩니다.



게시판을 추가할때 마다 똑같은 게시판소스를 가진 페이지를 올릴필요 없이 다중게시판은

말 그대로 DB테이블만 추가되고 게시판 소스를 가진 페이지는 게시판의 고유 ID를 가지고

여러의 게시판을 만들수 있다는 것입니다.(실제로는 게시판 소스를 가진 페이지는 하나면 됩니다.)


지금 지식인의 이러한 질문 게시판 또한 다중게시판 형식으로 만들었으며

지식인 같은 경우 수백만의 데이터가 존재하는데 매번 이러한 페이지를 추가 할수는 없습니다.

그래서 같은 소스를 가진 페이지를 쓰데 어떠한 고유의 ID를 가지고 서로 다른 디렉토리의 

게시판으로 사용 하는 것입니다.


정리해서 말하자면

일반 게시판은 

게시판소스페이지 : 데이타베이스 = 1 : 1 (게시판을 추가하려면 게시판에 해당하는 페이지를 전체 업로드)

다중 게시판은

게시판소스페이지 : 데이타베이스 = 1 : n (n,은 추가한 게시판의 갯수에 해당하는 DB테이블)


설명이 조금 이해가 되시련지 모르겠습니다.

그러나 게시판의 구조와 데이터베이스와의 관계를 생각하면 쉽게 알수 있습니다.
